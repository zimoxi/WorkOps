# M3 v0.6.0 Real Capability Roadmap

Version: 1.0

Date: 2026-07-04

Status: Ready

---

# Objective

定义 WorkOps 从 M2 Core Engine Foundation 进入 M3 Real Capability Integration。

M3 目标：将 Mock First 平台逐步连接真实能力。

---

# M2 Baseline

M2 已完成：

- Device（设备注册中心）
- Resource（资源注册中心）
- Operation（操作引擎）
- Task（任务引擎）
- Monitoring（监控引擎）
- Unified Store（统一数据源）
- Scheduler（调度引擎）
- History（执行历史）
- Component Library（组件库）

M2 架构必须保留，不能破坏。

---

# M3 Scope

## 1. Authentication（用户认证）

目标：

- 用户登录 / 登出
- 角色权限（Admin / Operator / Viewer）
- Session 管理
- 安全基础

Sprint 建议：

- Sprint013：Authentication Foundation

---

## 2. API Layer（接口层）

目标：

- Frontend → API → Service 架构
- RESTful API 设计
- 统一错误处理
- API 认证中间件

Sprint 建议：

- Sprint014：API Layer Foundation

---

## 3. Execution Layer（执行层）

目标：

- Task → Executor 架构
- 命令执行框架
- 执行结果收集
- 执行安全控制

Sprint 建议：

- Sprint015：Execution Layer Foundation

---

## 4. Device Adapter Layer（设备适配层）

目标：

- Device → Adapter 架构
- 统一设备连接接口
- 适配器注册机制
- 连接池管理

Sprint 建议：

- Sprint016：Device Adapter Foundation

---

## 5. Real Resource Discovery（真实资源发现）

目标：

- 真实设备资源扫描
- ZFS Pool / Dataset 发现
- 磁盘 / 分区识别
- 网络设备发现

Sprint 建议：

- Sprint017：Resource Discovery Foundation

---

## 6. Audit & Security（审计与安全）

目标：

- 操作审计日志
- 安全策略
- 敏感数据保护
- 权限审计

Sprint 建议：

- Sprint018：Audit & Security Foundation

---

# M3 禁止

不要一次性实现：

- SSH
- Docker
- PVE
- NAS
- Cloud

全部连接。

必须：一个 Sprint 一个能力。
