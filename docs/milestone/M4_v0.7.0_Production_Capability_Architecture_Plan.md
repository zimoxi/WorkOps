# 1. M4 总体目标

将 WorkOps 从 **M3 v0.6.0-alpha Internal Testing** 逐步推进到 **可受控连接真实设备的 Beta 系统**。

**核心原则：**
- Security First
- Read Only Before Write
- One Real Capability Per Sprint
- Backward Compatible
- No Generic Remote Command API

**M4 不等于直接连接所有真实设备。** 必须遵循渐进式安全接入策略。

# 3. Persistence Architecture

## 数据库接口

```python
class DatabaseRepository(ABC):
    """数据库 Repository 接口"""
    
    @abstractmethod
    def connect(self) -> bool:
        """连接数据库"""
        pass
    
    @abstractmethod
    def disconnect(self) -> None:
        """断开连接"""
        pass
    
    @abstractmethod
    def execute(self, query: str, params: dict) -> list:
        """执行查询"""
        pass
    
    @abstractmethod
    def migrate(self, version: str) -> bool:
        """执行迁移"""
        pass
```

## 持久化优先级

| 数据 | 优先级 | 说明 |
|------|--------|------|
| Session | High | 用户会话 |
| Task | High | 任务状态 |
| ExecutionResult | High | 执行结果 |
| Audit Log | High | 审计记录 |
| Credential | High | 凭据（加密） |
| Device | Medium | 设备信息 |
| Resource | Medium | 资源信息 |
| Operation | Low | 操作定义 |

## 迁移边界

- 每次只迁移一个模块
- 保留 MockRepository 作为回退
- 迁移前备份数据
- 迁移后验证完整性

class SecretProvider(ABC):
    """凭据提供者接口"""
    
    @abstractmethod
    def get_secret(self, credential_id: str) -> str:

# 5. Read-Only Real Device Connection Architecture

## SSHAdapter Read-Only 模式

```python
class SSHAdapter(BaseAdapter):
    """SSH Adapter（Read-Only）"""
    
    # 白名单命令（只读）
    ALLOWED_COMMANDS = {
        "uname -a",          # 系统信息
        "df -h",             # 磁盘使用
        "free -m",           # 内存使用
        "uptime",            # 运行时间
        "cat /proc/cpuinfo", # CPU 信息
    }
    
    def execute(self, command: str) -> dict:
        """执行命令（只允许白名单）"""
        if command not in self.ALLOWED_COMMANDS:
            raise CommandNotAllowedError(command)
        # 执行命令
        pass
```

## 连接流程

```
Authentication
    ↓
Permission Check
    ↓
Credential Lookup
    ↓
SSH Connection (with timeout)
    ↓
Host Key Verification
    ↓
Command Validation (whitelist only)
    ↓
Execute (read-only)
    ↓
Disconnect
    ↓
Audit Record
```

# 6. SSH Host Key Verification Strategy

## 策略选项

| 策略 | 说明 | 安全性 |
|------|------|--------|
| Strict | 首次连接后保存，后续验证 | 高 |
| Accept New | 接受新密钥，验证已知密钥 | 中 |
| None | 不验证 | 低（不推荐） |

## 实现

```python
class HostKeyPolicy:
    """主机密钥策略"""
    
    def __init__(self, policy_type: str = "strict"):
        self.policy_type = policy_type
        self.known_hosts = {}
    
    def verify(self, hostname: str, key: bytes) -> bool:
        """验证主机密钥"""
        if self.policy_type == "strict":
            return self._strict_verify(hostname, key)
        elif self.policy_type == "accept_new":
            return self._accept_new_verify(hostname, key)
        else:
            return True  # 不推荐
```

## 建议

- 生产环境：Strict
- 测试环境：Accept New
- 开发环境：Accept New 或 None（仅限本地）

# 7. Execution API Security Boundary

## 安全流程

```
API Request
    ↓
Authentication (Session Cookie)
    ↓
Permission Check (Admin/Operator)
    ↓
Request Validation (参数校验)
    ↓
Audit Record (记录请求)
    ↓
ExecutionService.execute_task()
    ↓
Adapter.execute()
    ↓
Audit Record (记录结果)
    ↓
API Response (脱敏)
```

## 禁止

| 禁止项 | 原因 |
|--------|------|
| 通用 shell command API | 安全风险 |
| 客户端直接提交任意命令 | 安全风险 |
| 跳过 Permission | 权限绕过 |
| 跳过 Validation | 注入风险 |
| 跳过 Audit | 审计缺失 |

# 9. Controlled Command Template Strategy

## 命令模板

```python
class CommandTemplate:
    """命令模板"""
    
    def __init__(self, name: str, template: str, params: list):
        self.name = name
        self.template = template  # "restic backup {path}"
        self.params = params      # ["path"]
    
    def render(self, **kwargs) -> str:
        """渲染命令"""
        # 验证参数
        for param in self.params:
            if param not in kwargs:
                raise MissingParameterError(param)
            # 验证参数值
            self._validate_param(param, kwargs[param])
        # 渲染
        return self.template.format(**kwargs)
```

## 参数白名单

```python
PARAM_VALIDATORS = {
    "path": lambda v: re.match(r'^/[\w/\-\.]+$', v),
    "repository": lambda v: re.match(r'^[\w\-\.]+$', v),
    "hostname": lambda v: re.match(r'^[\w\-\.]+$', v),
}
```

|--------|------|
| 任意命令 | 安全风险 |
| 参数注入 | 注入风险 |

# 18. Architecture Risks

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| SSH 连接不稳定 | 执行失败 | 超时、重试、连接清理 |
| 凭据泄漏 | 安全事故 | 加密存储、API 脱敏、日志脱敏 |
| 并发执行冲突 | 数据不一致 | 单实例锁、状态检查 |
| 数据库迁移失败 | 数据丢失 | 迁移边界、回滚方案 |
| 依赖安全漏洞 | 系统风险 | 依赖检查、版本锁定 |
| 主机密钥欺骗 | 中间人攻击 | Strict 主机密钥验证 |
| 命令注入 | 系统被入侵 | 命令模板、参数白名单 |
| 权限绕过 | 未授权访问 | 多层权限检查 |
