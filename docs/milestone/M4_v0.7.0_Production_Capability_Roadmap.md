# M4 v0.7.0 Production Capability Roadmap

Version: 1.0

Date: 2026-07-13

Status: Ready for Review

---

# 阶段目标

将 WorkOps 从：

```
M3 v0.6.0-alpha Internal Testing
```

逐步推进到：

```
可受控连接真实设备的 Beta 系统
```

M4 不等于直接连接所有真实设备。

必须遵循：

- Security First
- Read Only Before Write
- One Real Capability Per Sprint
- Backward Compatible
- No Generic Remote Command API

---

# Sprint 划分

## Sprint019 — Alpha Stabilization and Test Debt

**目标：**

- 解决 M3 Final Review 中的 15 个 HTTP API 测试失败
- 解决 1 个 Frontend 测试失败
- 建立可信的完整测试基线
- 整理重复 Technical Debt 编号
- 检查依赖、配置和运行目录

**禁止：**

- 连接真实设备
- 真实执行
- 新增业务能力

---

## Sprint020 — Persistence Foundation

**目标：**

- DatabaseRepository Interface
- 可替换的数据库实现基础
- Session 持久化方案
- Task 和 Result 持久化方案
- 数据迁移边界

**禁止：**

- 真实设备连接
- 真实任务执行
- 一次性迁移所有模块

---

## Sprint021 — Credential and Secret Management

**目标：**

- Credential Model
- Secret Provider Interface
- 加密存储边界
- API Response 脱敏
- 日志脱敏
- 配置模板

**禁止：**

- 凭据写入 Git
- 凭据写入普通日志
- 明文密码持久化
- 连接真实设备

---

## Sprint022 — SSH Adapter Read-Only Foundation

**目标：**

- 只实现 SSHAdapter
- 只允许连接测试
- 只允许白名单只读状态查询
- 超时与连接清理
- 主机密钥校验策略

**禁止：**

- 任意命令执行
- Backup
- Restore
- WinRM
- SNMP
- PVE/NAS/Cloud 全部同时接入

---

## Sprint023 — Execution API and Audit Boundary

**目标：**

- 受认证与权限保护的执行入口
- Request Validation
- Audit Record
- Idempotency 基础
- ExecutionService 接入边界

**禁止：**

- 通用 shell command API
- 客户端直接提交任意命令
- Scheduler 自动执行

---

## Sprint024 — Single Backup Executor

**目标：**

- 只实现一种受控备份能力
- 固定命令模板
- 参数白名单
- 路径校验
- 超时、取消和结果处理

**禁止：**

- 同时实现多个 Backup Provider
- 任意命令
- Restore

---

## Sprint025 — Scheduler Worker and Concurrency

**目标：**

- Worker
- 单实例锁
- 重复执行保护
- 超时
- 取消
- 崩溃恢复边界

**禁止：**

- 分布式 Worker 集群
- 未经审计的自动执行

---

## Sprint026 — History and Result Persistence

**目标：**

- ExecutionResult 持久化
- History 查询
- Task、Result、Audit 关联
- 分页和保留策略

---

## Sprint027 — Deployment and Upgrade Foundation

**目标：**

- Windows 部署方案
- Linux 部署方案
- 配置模板
- 数据目录规范
- 升级与回滚
- 数据迁移

---

## Sprint028 — Security Hardening and Beta Review

**目标：**

- CSRF
- Secure Cookie
- 登录频率限制
- Session 安全
- 日志与错误脱敏
- 权限复核
- 依赖安全检查
- Beta Final Review

---

# 强制约束

1. M4 开始前仍禁止真实设备连接。

2. Sprint019—021 不允许连接真实设备。

3. Sprint022 只允许 SSH Read Only。

4. 不允许一次实现：

```
SSH
WinRM
SNMP
PVE
NAS
Cloud
```

5. 不允许提供任意命令执行接口。

6. Task 仍是唯一执行入口。

7. Operation 不得直接调用 Adapter。

8. 所有真实操作必须：

```
Authentication
    ↓
Permission
    ↓
Validation
    ↓
Audit
    ↓
ExecutionService
    ↓
Adapter
```

9. 每个 Sprint 完成后必须执行：

```
Architecture Review
    ↓
Coding
    ↓
Test
    ↓
Completion Review
    ↓
Git Commit
    ↓
Git Push
    ↓
必要时 Tag
```

10. 必须保持 M2 和 M3 向后兼容。

# 风险分析

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| SSH 连接不稳定 | 执行失败 | 超时、重试、连接清理 |
| 凭据泄漏 | 安全事故 | 加密存储、API 脱敏、日志脱敏 |
| 并发执行冲突 | 数据不一致 | 单实例锁、状态检查 |
| 数据库迁移失败 | 数据丢失 | 迁移边界、回滚方案 |
| 依赖安全漏洞 | 系统风险 | 依赖检查、版本锁定 |

# 成功标准

M4 完成后，系统应满足：

1. 可通过 SSH 连接真实设备
2. 可执行受控备份操作
3. 所有操作有审计记录
4. 凭据安全存储
5. 支持 Windows/Linux 部署
6. 通过 Beta Final Review
7. M2/M3 向后兼容
